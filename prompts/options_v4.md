# 关键立场提醒（评估的第一原则）

# 第 0 步：输入有效性预检（最先执行，不通过则短路）
若 [OUTPUT] 为空、无法解析出任何选项文本、或选项数为 0，则**不要**继续评分（不要把缺失当作"合格/PASS"）。直接输出：gate 全部 "FAIL"、quality 全部 "差"、overall=1、verdict 写明"未生成有效选项"，并结束。只有当存在至少一个可读选项时，才进入下面的步骤。

# 评估步骤
先逐条审视每个选项是否越界，再做维度判断（reasoning 在前），最后给结论。

## 第一步：逐选项边界审查
对每个选项判断：是否只用了玩家此刻已知信息？是否承接了本轮剧情？

## 第二步：门禁（每项 PASS / FAIL，给引证）
G1 玩家信息边界：是否有任一选项用全知信息、泄露未揭晓伏笔/NPC 隐藏动机/未来走向，或让玩家打听其身份本就知晓的常识？（有=FAIL）
G2 承接本轮剧情：选项是否衔接刚展示的叙事（呼应其中人物/信息/线索）？（凭空跳出=FAIL）

## 第三步：质量维度（每项 差 / 合格 / 好，给引证）
Q1 角色一致性：是否符合主角人设与当前处境（而非通用空选项）。
Q2 多样性：多个选项是否覆盖不同意图/策略（如追问/推进/迂回），而非同义改写。
Q3 可玩性表达：是否简洁、像玩家真实会说/做、有代入感与钩子。
Q4 数量与覆盖：数量是否在 1–4、是否覆盖本轮关键决策面与不同动作类型。

# overall 合成规则（严格照此推导）
- 若门禁全 PASS：设"好"数量 g、"差"数量 b（合格不计）。overall=5（b=0且g≥2）；=4（b=0）；=3（b≤1）；=3（其余）。
- 若有门禁 FAIL：overall=2（仅1项越界且影响有限）；=1（多条越界或严重泄露伏笔）。
